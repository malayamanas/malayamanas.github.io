Short Description An adversary targets credentials or authentication material to enable further compromise by exploiting OS Credential Dumping: LSA Secrets. Windows Security Event 4692 (Backup of data protection master key was attempted) provides the primary detection signal when properly audited and monitored in a SIEM.

2. MITRE ATT&CK Mapping

  • Tactic : Credential Access

Technique : OS Credential Dumping: LSA Secrets (T1003.004)

3. How the Attack Typically Works (Procedure)

  1. Attacker achieves local administrator or SYSTEM access on a targeted host or across the domain.
  2. Volume Shadow Copy deletion is executed using vssadmin delete shadows /all /quiet, wmic shadowcopy delete, or bcdedit to disable recovery.
  3. These commands appear as Event 4688 process creation events with the recognisable command lines targeting VSS.
  4. Simultaneous Event 7036 shows backup services (Windows Backup, Volume Shadow Copy Service, VSS) stopping.
  5. Without shadow copies, recovery from ransomware encryption or data destruction is severely hampered.
Most common tools observed in 2024–2026:
  • vssadmin.exe (native)
  • wmic.exe shadowcopy delete (native)
  • PowerShell Get-WmiObject Win32_ShadowCopy | Remove-WmiObject
  • Ransomware pre-encryption routines (LockBit, Conti, BlackCat)
  • bcdedit /set {default} recoveryenabled No

5. Detection / Hunting Recommendations

Strong / Classic Signature Rule (Medium Confidence)
  • Alert on Event ID 4692 matching the highest-fidelity field combination shown above
  • Apply thresholds appropriate to the environment: a single event may be sufficient for critical-level detections; volume thresholds apply for medium/low
  • Correlate with Event 4624 (successful logon) using SubjectLogonId to establish the session context and identify the source IP and workstation
Highest-Fidelity Correlation (SIEM Query Logic): 
EventID = 4692
AND SubjectUserName NOT IN <approved_admin_accounts>
Contextual / Behavioral Hunting Queries (SIEM / EDR)
  • Correlate 4692 events where SubjectLogonId links to a Type 3 (network) or Type 10 (remote interactive) logon from an external IP address
  • Alert on 4692 events from accounts in privileged groups (Domain Admins, Enterprise Admins) outside approved maintenance windows
  • Hunt for 4692 events followed within 60 seconds by lateral movement indicators (5140, 4624 to other hosts) from the same SubjectLogonId
False Positive Considerations
  • Legitimate administrative activity during change management windows may generate similar events
  • Automated enterprise management platforms (SCCM, Ansible, Puppet) perform similar operations as part of configuration management
  • Security scanning and vulnerability assessment tools may trigger events matching this pattern
Recommended Hardening (Prevention & Impact Reduction)
  • Apply the Principle of Least Privilege: restrict the ability to perform this action to explicitly approved accounts
  • Enable advanced auditing via Group Policy (Advanced Audit Policy Configuration) to ensure all relevant events are captured
  • Deploy a SIEM with behavioral analytics to baseline normal activity and alert on deviations
  • Implement privileged access management (PAM) to control and audit use of privileged credentials

6. Recommended Response Actions

  • Isolate the affected host from the network immediately if active exploitation or lateral movement is confirmed.
  • Identify the initiating account (SubjectUserName/SubjectLogonId) and disable it pending investigation.
  • Preserve all relevant Windows Security Event logs from the affected host(s) before any remediation.
  • Correlate the triggering event with Event 4624 logon records to identify the source IP, workstation, and session that performed the action.
  • Reset passwords for all accounts confirmed or suspected to be compromised; prioritise privileged accounts.
  • If KRBTGT hash was exposed (DCSync), perform two sequential KRBTGT password resets to invalidate existing golden tickets.
  • Audit and revoke any Kerberos service tickets (TGTs/TGS) that may have been captured; force re-authentication.
  • Review and revoke any certificates issued around the time of the attack (check AD CS audit logs).
  • Hunt for pass-the-hash or pass-the-ticket usage across the environment using Event 4624 NTLM logon correlation.
