Short Description An adversary targets credentials or authentication material to enable further compromise by exploiting OS Credential Dumping: DCSync. Windows Security Event 4662 (An operation was performed on an object) provides the primary detection signal when properly audited and monitored in a SIEM.

2. MITRE ATT&CK Mapping

  • Tactic : Credential Access

Technique : OS Credential Dumping: DCSync (T1003.006)

3. How the Attack Typically Works (Procedure)

  1. Attacker gains Domain Admin, Enterprise Admin, or accounts with explicit directory replication rights.
  2. Attacker uses Mimikatz lsadump::dcsync, Impacket secretsdump.py -just-dc, or DSInternals to request domain replication.
  3. Tool issues MS-DRSR GetNCChanges requests to a legitimate DC, presenting stolen credentials with replication rights.
  4. DC responds with password attributes (unicodePwd, lmPwdHistory, ntPwdHistory, supplementalCredentials) for targeted accounts.
  5. Extracted hashes enable pass-the-hash, pass-the-ticket, golden ticket creation, or offline cracking.
Most common tools observed in 2024–2026:
  • Mimikatz lsadump::dcsync
  • Impacket secretsdump.py
  • DSInternals Get-ADReplAccount
  • Rubeus dcsync
  • SharpKatz

5. Detection / Hunting Recommendations

Strong / Classic Signature Rule (Critical Confidence)
  • Alert on Event ID 4662 matching the highest-fidelity field combination shown above
  • Apply thresholds appropriate to the environment: a single event may be sufficient for critical-level detections; volume thresholds apply for medium/low
  • Correlate with Event 4624 (successful logon) using SubjectLogonId to establish the session context and identify the source IP and workstation
Highest-Fidelity Correlation (SIEM Query Logic): 
EventID = 4662
AND SubjectUserName NOT IN <approved_admin_accounts>
Contextual / Behavioral Hunting Queries (SIEM / EDR)
  • Correlate 4662 events where SubjectLogonId links to a Type 3 (network) or Type 10 (remote interactive) logon from an external IP address
  • Alert on 4662 events from accounts in privileged groups (Domain Admins, Enterprise Admins) outside approved maintenance windows
  • Hunt for 4662 events followed within 60 seconds by lateral movement indicators (5140, 4624 to other hosts) from the same SubjectLogonId
False Positive Considerations
  • Azure AD Connect / MSOL_ sync accounts legitimately perform directory replication
  • Read-Only Domain Controller (RODC) replication from full DCs generates similar events
  • ADMT (Active Directory Migration Tool) requires replication rights for domain migration tasks
Recommended Hardening (Prevention & Impact Reduction)
  • Audit and remove unnecessary Replicating Directory Changes permissions — only DCs and approved sync accounts should hold these
  • Enable LDAP signing and LDAP channel binding to prevent credential relay attacks that could lead to DCSync
  • Monitor for new grants of replication rights via Event 5136 (attribute change to ACL on domain object)
  • Use tiered administration model: no Domain Admins logging on to workstations

6. Recommended Response Actions

  • Isolate the affected host from the network immediately if active exploitation or lateral movement is confirmed.
  • Identify the initiating account (SubjectUserName/SubjectLogonId) and disable it pending investigation.
  • Preserve all relevant Windows Security Event logs from the affected host(s) before any remediation.
  • Correlate the triggering event with Event 4624 logon records to identify the source IP, workstation, and session that performed the action.
  • Reset passwords for all accounts confirmed or suspected to be compromised; prioritise privileged accounts.
  • If KRBTGT hash was exposed (DCSync), perform two sequential KRBTGT password resets to invalidate existing golden tickets.
  • Audit and revoke any Kerberos service tickets (TGTs/TGS) that may have been captured; force re-authentication.
  • Review and revoke any certificates issued around the time of the attack (check AD CS audit logs).
  • Hunt for pass-the-hash or pass-the-ticket usage across the environment using Event 4624 NTLM logon correlation.
