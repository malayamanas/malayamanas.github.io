Short Description An adversary bypasses, disables, or subverts security monitoring and controls by exploiting Impair Defenses: Disable or Modify Tools. Windows Security Event 4738 (A user account was changed) provides the primary detection signal when properly audited and monitored in a SIEM.

2. MITRE ATT&CK Mapping

  • Tactic : Defense Evasion

Technique : Impair Defenses: Disable or Modify Tools (T1562.001)

3. How the Attack Typically Works (Procedure)

  1. Attacker enumerates domain accounts with the DONT_REQUIRE_PREAUTH userAccountControl flag set (via LDAP or BloodHound).
  2. AS-REQ (Authentication Service Request) is sent to the KDC without pre-authentication for these accounts.
  3. KDC responds with an AS-REP containing the account's TGT encrypted with the user's password hash.
  4. The encrypted TGT portion is exported and cracked offline using hashcat or John the Ripper without any successful authentication.
  5. Event 4768 with pre-authentication type 0 (none) from a non-DC source reveals the enumeration attempt.
Most common tools observed in 2024–2026:
  • Rubeus (asreproast command)
  • Impacket GetNPUsers.py
  • PowerSploit Get-DomainUser -PreauthNotRequired
  • CrackMapExec --asreproast
  • Kerbrute

5. Detection / Hunting Recommendations

Strong / Classic Signature Rule (High Confidence)
  • Alert on Event ID 4738 matching the highest-fidelity field combination shown above
  • Apply thresholds appropriate to the environment: a single event may be sufficient for critical-level detections; volume thresholds apply for medium/low
  • Correlate with Event 4624 (successful logon) using SubjectLogonId to establish the session context and identify the source IP and workstation
Highest-Fidelity Correlation (SIEM Query Logic): 
EventID = 4738
AND SubjectUserName NOT IN <approved_admin_accounts>
Contextual / Behavioral Hunting Queries (SIEM / EDR)
  • Correlate 4738 events where SubjectLogonId links to a Type 3 (network) or Type 10 (remote interactive) logon from an external IP address
  • Alert on 4738 events from accounts in privileged groups (Domain Admins, Enterprise Admins) outside approved maintenance windows
  • Hunt for 4738 events followed within 60 seconds by lateral movement indicators (5140, 4624 to other hosts) from the same SubjectLogonId
False Positive Considerations
  • Legitimate disk encryption tools (BitLocker, VeraCrypt, enterprise DLP encryption) generate similar file access patterns
  • Large-scale data migration or backup operations may create bursts of file creation/modification events
  • Anti-virus scan engines performing deep reads of files create access events without modification
Recommended Hardening (Prevention & Impact Reduction)
  • Implement immutable offline or air-gapped backups following the 3-2-1-1-0 backup rule
  • Deploy application allowlisting to prevent unknown executables from running on endpoints and servers
  • Enable Controlled Folder Access (Windows Defender) on all endpoints to block unauthorized file modification
  • Segment networks to prevent ransomware from reaching backup infrastructure and file servers simultaneously

6. Recommended Response Actions

  • Isolate the affected host from the network immediately if active exploitation or lateral movement is confirmed.
  • Identify the initiating account (SubjectUserName/SubjectLogonId) and disable it pending investigation.
  • Preserve all relevant Windows Security Event logs from the affected host(s) before any remediation.
  • Correlate the triggering event with Event 4624 logon records to identify the source IP, workstation, and session that performed the action.
  • Reset passwords for all accounts confirmed or suspected to be compromised; prioritise privileged accounts.
  • If KRBTGT hash was exposed (DCSync), perform two sequential KRBTGT password resets to invalidate existing golden tickets.
  • Audit and revoke any Kerberos service tickets (TGTs/TGS) that may have been captured; force re-authentication.
  • Review and revoke any certificates issued around the time of the attack (check AD CS audit logs).
  • Hunt for pass-the-hash or pass-the-ticket usage across the environment using Event 4624 NTLM logon correlation.
