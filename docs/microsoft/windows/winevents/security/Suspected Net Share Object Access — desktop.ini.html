Short Description An adversary gathers sensitive data from the target environment for exfiltration by exploiting Data from Network Shared Drive. Windows Security Event 5145 (A network share object was checked to see whether client can be granted desired access) provides the primary detection signal when properly audited and monitored in a SIEM.

3. How the Attack Typically Works (Procedure)

  1. Attacker achieves initial access to the target environment through social engineering, exploitation, or use of stolen credentials.
  2. Using the foothold, the attacker performs collection, exfiltration activity related to Suspected Net Share Object Access — desktop.ini, targeting the relevant Windows subsystem or service.
  3. The attack leverages MITRE ATT&CK technique T1039 to achieve the objective, using tools and techniques common to this attack category.
  4. Windows Security Event(s) 5145 are generated, capturing critical details of the activity including the actor identity, target object, and operation performed.
  5. Post-attack, the attacker may perform additional lateral movement, persistence, or data collection based on the access gained.
Most common tools observed in 2024–2026:
  • CobaltStrike (commercial C2 framework)
  • Metasploit Framework (open-source exploitation)
  • Impacket (Python network protocol library)
  • CrackMapExec (post-exploitation framework)
  • PowerSploit / PowerView (PowerShell attack modules)

5. Detection / Hunting Recommendations

Strong / Classic Signature Rule (Medium Confidence)
  • Alert on Event ID 5145 matching the highest-fidelity field combination shown above
  • Apply thresholds appropriate to the environment: a single event may be sufficient for critical-level detections; volume thresholds apply for medium/low
  • Correlate with Event 4624 (successful logon) using SubjectLogonId to establish the session context and identify the source IP and workstation
Highest-Fidelity Correlation (SIEM Query Logic): 
EventID = 5145
AND SubjectUserName NOT IN <approved_admin_accounts>
AND IpAddress NOT IN <trusted_ip_ranges>
Contextual / Behavioral Hunting Queries (SIEM / EDR)
  • Correlate 5145 events where SubjectLogonId links to a Type 3 (network) or Type 10 (remote interactive) logon from an external IP address
  • Alert on 5145 events from accounts in privileged groups (Domain Admins, Enterprise Admins) outside approved maintenance windows
  • Hunt for 5145 events followed within 60 seconds by lateral movement indicators (5140, 4624 to other hosts) from the same SubjectLogonId
False Positive Considerations
  • File servers serving high volumes of legitimate requests will generate many 5140/5145 events
  • Backup systems accessing administrative shares (ADMIN$, C$) for agent deployment
  • IT automation platforms using SMB for software distribution
  • Domain controller replication and SYSVOL access generates 5140 events from all domain-joined machines
Recommended Hardening (Prevention & Impact Reduction)
  • Disable SMBv1 across the entire environment (critical for preventing relay attacks and EternalBlue exploitation)
  • Enable SMB signing to prevent relay attacks that don't require cracked credentials
  • Block direct SMB (445) at the network perimeter and between network segments where SMB is not required
  • Apply firewall rules restricting ADMIN$ and C$ access to management VLAN IP ranges only

6. Recommended Response Actions

  • Isolate the affected host from the network immediately if active exploitation or lateral movement is confirmed.
  • Identify the initiating account (SubjectUserName/SubjectLogonId) and disable it pending investigation.
  • Preserve all relevant Windows Security Event logs from the affected host(s) before any remediation.
  • Correlate the triggering event with Event 4624 logon records to identify the source IP, workstation, and session that performed the action.
  • Review all events from the same SubjectLogonId session to identify what other actions the attacker performed.
  • Check for additional persistence mechanisms or privilege escalation attempts in the same time window.
  • Conduct a threat hunt across similar hosts in the same network segment for the same indicators.
  • Engage the incident response team to scope the full extent of the compromise before returning systems to production.
