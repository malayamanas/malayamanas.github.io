Short Description Brute-force password guessing against a single target account — most commonly domain accounts, local accounts, or service accounts — via repeated failed authentication attempts. Attackers systematically try passwords (dictionary, incremental, credential stuffing from previous breaches) until success or lockout. High-volume attempts against one account are easier to detect than low-and-slow spraying.

2. MITRE ATT&CK Mapping

  • Tactic : Credential Access

Technique : Brute Force: Password Guessing / Password Spraying (T1110.001)

3. How the Attack Typically Works (Procedure)

  1. Attacker identifies valid usernames (via enumeration, leaks, OSINT, LDAP queries, or previous compromise).
  2. Attacker selects one high-value target account (e.g., Administrator, service account, executive, shared mailbox).
  3. Attacker launches automated password guessing:

  • Dictionary attacks (common passwords, seasonal variants)
  • Incremental / mask attacks (e.g., Password1 → Password2)
  • Credential stuffing (reusing known breached passwords)

  1. Attempts occur over:

  • SMB (445), RDP (3389), WinRM (5985/5986), LDAP (389/636), HTTP/NTLM endpoints
  • Tools: Hydra, Medusa, CrackMapExec, Patator, custom scripts

  1. Goal: successful logon (4624) or account lockout (4740) to confirm weak password.

Most common tools & vectors observed in 2024–2026:
  • Hydra / Medusa / Ncrack
  • CrackMapExec (-u user -p passwords.txt)
  • Patator / crowbar
  • RDP brute-forcing (xHydra, custom RDP clients)
  • SMB / WinRM brute-forcing (impacket, evil-winrm)
  • External attacks against exposed RDP / VPN / OWA / ADFS endpoints

Event ID = 4625

5. Detection / Hunting Recommendations

Strong / Classic Signature Rule (High Confidence)
  • Event ID = 4625
  • Same TargetUserName appears ≥ 15–25 times in short window (5–30 min)
  • Status/SubStatus = bad password combo (0xC000006D/0xC000006A)
  • IpAddress is external OR not in corporate VPN range
  • Exclude known noisy scanners / monitoring tools via allow-list
Behavioral / Correlation-based Hunting Queries (SIEM / EDR)
  • Count of 4625 events grouped by TargetUserName → high count + low unique IpAddress = focused brute force
  • 4625 burst against privileged accounts (Administrator, krbtgt, service accounts)
  • 4625 + same IpAddress → successful 4624 shortly after = successful brute force
  • Correlate with preceding enumeration (4661/4662 LDAP user reads, 4798/4799 local group enumeration)
False Positive Considerations
  • Users mistyping passwords repeatedly (usually low volume, same workstation)
  • Legacy applications / scripts with wrong credentials
  • Penetration testing / red-team activity (should be coordinated)
  • Backup / monitoring agents with stale credentials
Recommended Hardening (Prevention & Impact Reduction)
  • Enforce strong account lockout policy (5–10 attempts, 15–30 min lockout)
  • Implement multi-factor authentication (especially for RDP, VPN, OWA, ADFS)
  • Disable NTLM where possible (especially externally)
  • Block direct internet exposure of RDP (3389), SMB (445), WinRM (5985/5986)
  • Use Azure AD Password Protection or on-prem equivalent for banned password lists
  • Monitor and alert on external 4625 volume against any single account

6. Recommended Response Actions

  • Temporarily lock or disable the targeted account if brute force is ongoing
  • Block source IP(s) at firewall / endpoint level
  • Force password reset + MFA re-enrollment for targeted account
  • Hunt for successful logon immediately after failures (4624 with same IpAddress)
  • Review 4625, 4624, and 4771 (Kerberos pre-auth failures) events from the source IP
  • Check for follow-on activity: lateral movement (5140/5145 shares, 7045 services), privilege escalation (4672, 5136)
