Short Description An adversary bypasses, disables, or subverts security monitoring and controls through Suspicious Process Spawned by svchost — Hollow Process. Sysmon Event(s) 1, 8, 25 (Process Creation, CreateRemoteThread, ProcessTampering) provide the primary Sysmon-layer detection signal for this technique.

2. MITRE ATT&CK Mapping

  • Tactic : Defense Evasion

Technique : T1055.012 (T1055.012)

3. How the Attack Typically Works (Procedure)

  1. Attacker has code execution and targets a legitimate host process to inject into for stealth or memory-resident execution.
  2. Sysmon Event 10 (ProcessAccess) fires when the attacker's process opens a handle to the target process — GrantedAccess 0x001F0FFF or 0x0020 (VM_WRITE) indicate injection intent.
  3. Sysmon Event 8 (CreateRemoteThread) fires when a new thread is injected into the target process — StartModule=NULL indicates shellcode (no DLL on disk).
  4. For process hollowing, Sysmon Event 25 (ProcessTampering) fires when Sysmon detects the in-memory image differs from the on-disk binary.
  5. Sysmon Event 3 from the now-compromised target process reveals C2 connections that appear to originate from the legitimate process.
Most common tools observed in 2024–2026:
  • Cobalt Strike (reflective DLL injection, process injection modules)
  • Meterpreter (migrate command for process injection)
  • Custom shellcode loaders
  • Process hollowing frameworks (Donut, Gargoyle)

4. Key Sysmon Event Indicators

Sysmon Event 1 — Process Creation

A new process was created. Captures image path, command line, parent, hashes, user, and integrity level.

FieldSuspicious Value / PatternWhy It Matters
ImageNon-system path (%TEMP%, %AppData%, %ProgramData%), renamed system binaries, known tool names (mimikatz.exe, rubeus.exe, psexec.exe, cobalt*)Full path of the spawned executable; non-standard paths and known attacker tools are primary IOCs
CommandLineEncoded commands (-EncodedCommand, -enc), LOLBin abuse flags (certutil -decode, mshta http://), credential access arguments (sekurlsa, lsadump, pth)Full command line reveals exact attacker intent; encoded or obfuscated arguments are critical IOCs
ParentImageOffice apps (winword.exe, excel.exe) spawning PowerShell/cmd; browsers spawning cmd; svchost.exe spawning unexpected children; w3wp.exe (IIS) spawning shellsAbnormal parent-child relationships reveal the initial execution vector (macro, exploit, injection)
ParentCommandLineEncoded PowerShell from Office; mshta.exe launching script; wscript.exe executing remote contentParent's full command clarifies the delivery mechanism and payload stage
OriginalFileNameMismatch between Image filename and OriginalFileName field (e.g., Image=svchost.exe but OriginalFileName=mimikatz.exe)Catches renamed attacker tools that attempt to masquerade as legitimate Windows binaries
HashesKnown malware hashes (cross-reference threat intelligence feeds); unsigned binaries with high-entropy namesHash-based detection is immune to filename changes; enables instant enrichment with threat intelligence
IntegrityLevelHigh or System from a process spawned by a low-privilege parent; elevation without UAC prompt contextIndicates whether the process ran with elevated privileges; unexpected elevation reveals bypass techniques
UserSYSTEM-context processes from interactive user sessions; service accounts launching interactive processesProcess owner; unexpected user context for a given image path indicates impersonation or privilege escalation

Sysmon Event 8 — CreateRemoteThread

A thread was created in another process. Primary indicator of process injection and shellcode injection.

FieldSuspicious Value / PatternWhy It Matters
TargetImagelsass.exe (credential theft), explorer.exe (process hollowing C2), svchost.exe (service-based injection), browsers (credential/session theft)Process being injected into; high-value target processes are chosen for stealth and access to sensitive memory
SourceImageUnexpected source: powershell.exe, cmd.exe, Office applications, wscript.exe injecting into system processesProcess performing the injection; non-standard source processes performing CreateRemoteThread reveal the attacker's loader
StartModuleNULL (shellcode, no module name) or a module in %TEMP%/%AppData%; known injection staging DLLsModule containing the injected thread entry point; NULL StartModule is a classic shellcode injection indicator
StartFunctionLoadLibraryA/W with a malicious DLL argument (reflective loading); CreateThread shellcode stubsEntry point of the injected thread; LoadLibrary calls with suspicious argument paths reveal DLL injection

Sysmon Event 25 — ProcessTampering

A process image was tampered (process hollowing, doppelgänging). Detected by Sysmon comparing on-disk vs in-memory image.

FieldSuspicious Value / PatternWhy It Matters
ImageSystem processes (svchost.exe, explorer.exe, taskhost.exe) that have been hollowed; any process where Sysmon detects a mismatch between the on-disk binary and the in-memory imageProcess whose image was tampered; system process names are deliberately chosen to evade detection — this event catches the hollowing despite the legitimate-looking name
TypeProcessHollowing (SuspendResumeInjection), ProcessDoppelgänging (transactional NTFS injection), Herpaderping (file-on-disk replaced after mapping)Specific tampering technique detected; different types have different forensic artefacts and require different response procedures

5. Detection / Hunting Recommendations

Strong / Classic Signature Rule (High Confidence)
  • Alert on Sysmon event(s) 1, 8, 25 matching the field combinations shown in Section 4
  • Apply environment-specific thresholds: a single matching event may be sufficient for critical-level detections
Sigma Detection Rule (YAML skeleton): 
title: Suspicious Process Spawned by svchost — Hollow Process
status: experimental
logsource:
    category: process_creation  # adjust per primary Sysmon event type
    product: windows
detection:
    selection:
    EventID: 1  # Sysmon Process Creation
    ParentImage|endswith|any:  # Suspicious parent
      - '\\winword.exe'
      - '\\excel.exe'
      - '\\powerpnt.exe'
      - '\\w3wp.exe'
    Image|endswith|any:
      - '\\cmd.exe'
      - '\\powershell.exe'
      - '\\wscript.exe'
    condition: selection
level: high
tags:
    - defense_evasion
    - t1055.012
Contextual / Behavioral Hunting Queries (SIEM / EDR)
  • Hunt for Sysmon 1 / Sysmon 8 / Sysmon 25 events where Image is NOT in the approved software baseline
  • Alert on Sysmon 1 / Sysmon 8 / Sysmon 25 from accounts in privileged groups outside approved maintenance windows
  • Correlate Sysmon 1 events with subsequent Windows Security 4624/4625 events to link process activity to authentication events
  • Stack-rank Sysmon 1 / Sysmon 8 / Sysmon 25 events by Image path to identify rare/novel executables across the fleet
False Positive Considerations
  • Legitimate administrative operations by IT staff may generate similar Sysmon events
  • Authorized enterprise management platforms performing identical operations
  • Security scanning and vulnerability assessment tools in approved assessment windows
  • Development and testing environments running attack simulation frameworks
Recommended Hardening (Prevention & Impact Reduction)
  • Apply Principle of Least Privilege: restrict the ability to perform this attack to the minimum required accounts
  • Deploy comprehensive Sysmon configuration (SwiftOnSecurity or Florian Roth config) to capture all relevant events
  • Configure SIEM correlation rules to baseline normal Sysmon patterns and alert on deviations
  • Enable Microsoft Defender for Endpoint or equivalent EDR for behavioral detection in addition to Sysmon event-based detection

6. Recommended Response Actions

  • Isolate the affected host from the network immediately if active exploitation or lateral movement is confirmed.
  • Identify the initiating account (from Sysmon Event 1 User field or Windows Security 4624 correlation) and disable it pending investigation.
  • Preserve Sysmon event logs from the affected host(s) before any remediation — export the Microsoft-Windows-Sysmon/Operational channel.
  • Correlate the Sysmon events with Windows Security logs using the LogonId field to link process activity to authenticated sessions.
  • Perform memory analysis of the affected system to identify all injected code, hollowed processes, and loaded kernel modules.
  • Search for the same driver hash or injected DLL hash across all endpoints using EDR retrospective hunt.
  • If a vulnerable driver was loaded (BYOVD), check all systems for the vulnerable driver and remove/block it via WDAC policy.
  • Rebuild the affected system — kernel-level compromise cannot be reliably remediated without full OS reinstall.
